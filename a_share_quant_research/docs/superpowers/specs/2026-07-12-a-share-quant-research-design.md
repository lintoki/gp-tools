# A股半自动量化投研系统设计规格

## 1. 需求分析

### 1.1 目标

系统每天以可配置的A股主板历史股票池为范围，读取并验证真实数据，计算可解释因子，结合行业、公告、新闻、美国市场环境和期货风险资源，最多输出5只研究候选。系统只提供研究候选、证据和风险分析，不连接券商、不自动下单、不承诺收益。

第一版只使用日线和低频策略，不使用深度学习，不开发高频交易，不生成美股或期货交易信号。

### 1.2 成功标准

1. 每个生产数据字段都携带 `source_name`、`source_uri`、`fetched_at`、`effective_at`、`available_at`、`run_id` 和内容哈希。
2. 必需数据未通过完整性、时效性、模式、一致性或披露时点检查时，整次研究运行在任何评分或语言模型推理前终止。
3. 生产代码不存在随机数据、示例数据、固定占位值或静默缓存回退。测试夹具只能存在于 `tests/fixtures`，并带 `TEST_ONLY` 标记。
4. 所有候选能够追溯到数据快照、股票池判断、因子原始值、标准化值、权重、证据和历史实验。
5. 回测处理复权、停牌、涨跌停、T+1、费用、印花税、滑点、披露日期、历史股票池、无法成交和未来函数。
6. 每日报告同时输出 Markdown、JSON 和 CSV，允许输出0只并明确写“不推荐”。
7. 手动命令稳定且全量测试通过后，才创建每日研究 Skill。

### 1.3 非目标

- 券商连接、自动交易、自动套保、账户或持仓控制。
- 高频、分钟级撮合、深度学习、强化学习和复杂机器学习。
- 美股选股、美国证券交易建议、期货交易建议。
- 保证免费外部数据源永久可用或保证每天都有候选。
- 在数据缺失时由语言模型补全、估算或猜测。

## 2. 已选架构

采用完全自研生产架构。数据、因子、股票池、事件验证、撮合、回测、实验账本和报告均由本项目实现。Qlib不进入生产运行时；它只允许作为可选的测试对照工具，用于发现自研回测偏差。

新项目位于 `a_share_quant_research/`，不修改或继承现有 `shortline_quant` 的数据路径。原因是现有实现包含吞异常、固定换手率、固定市值和示例数据回退，与本项目失败关闭原则冲突。

### 2.1 数据流

```text
公共主源/备用源
    -> 有限重试采集器
    -> raw 原始响应与请求清单
    -> 标准化器
    -> normalized Parquet
    -> 运行级 Data Quality Gate
        -> 失败: 终止，输出错误清单，不进入评分/LLM
        -> 通过: 生成 curated DuckDB 快照与 manifest
    -> 历史股票池 + 因子引擎 + Evidence Gate + 外部环境
    -> 候选决策
    -> 自研事件驱动回测与历史信号统计
    -> Markdown / JSON / CSV 报告
```

### 2.2 模块边界

| 模块 | 责任 | 禁止事项 |
|---|---|---|
| `providers` | 获取真实外部数据并声明能力 | 评分、静默降级、伪造字段 |
| `storage` | raw/normalized/curated 分层、快照和血缘 | 自动替换过期数据 |
| `quality` | 模式、完整率、时效、唯一性、范围、跨源校验 | 将失败改成警告后继续 |
| `universe` | 按历史日期重建可交易股票池 | 使用今天的股票名单回填历史 |
| `factors` | 确定性公式、标准化、权重评分 | 调用语言模型计算数值 |
| `evidence` | 分级、去重、独立来源、矛盾证据和验证状态 | D级证据单独形成理由 |
| `context` | 美国行业、股市趋势、利率、波动率、期货持仓风险 | 生成境外交易信号 |
| `backtest` | 事件驱动撮合、成本、PIT、指标和实验留档 | 删除失败实验或只展示最优参数 |
| `reporting` | 从同一结构化对象渲染三种格式 | 自行补充未计算的数据 |
| `cli` | 串联可审计的手动阶段 | 绕过门禁直接生成报告 |

## 3. 数据源和提供者接口

### 3.1 第一版免费源

| 数据域 | 主源 | 备用/交叉校验 | 生产要求 |
|---|---|---|---|
| A股日线、复权、基础信息、财务 | AkShare 对应公开接口 | 可配置第二 AkShare 上游或导入的官方文件 | 必须记录实际上游名称和接口参数 |
| 公司公告、业绩预告、快报 | 上交所、深交所、巨潮公开信息，经直接或 AkShare 适配 | 另一官方站点 | 核心事件必须回链官方原文 |
| A股新闻 | 可靠财经媒体公开内容，经 AkShare/HTTP/RSS 适配 | 第二独立媒体 | 转载不计为独立来源 |
| 行业和市场基准 | 中证/交易所公开指数，经适配器读取 | 第二行情来源 | 与个股交易日历一致 |
| 美国公司和行业证据 | SEC EDGAR、BEA、BLS、政策机构公开材料 | 第二官方来源 | 只作为A股外部环境 |
| 美国市场状态 | FRED公开序列及可配置公开指数源 | 第二指数源 | 交易日历感知时效 |
| 期货风险资源 | CFTC COT、公开期限结构或指数序列 | 第二公开来源 | 只输出风险标签，不下单 |

免费源不可用时系统可以失败，但不可以猜测。主源和备用源都失败时，运行状态为 `BLOCKED_DATA`。

### 3.2 统一接口

所有 provider 实现以下协议，并返回 `DataBatch`：

```python
class DataProvider(Protocol):
    name: str
    capabilities: frozenset[DatasetKind]

    def fetch(self, request: FetchRequest) -> DataBatch: ...
```

`FetchRequest` 包含数据域、标的、开始/结束日期、截至时点和 `run_id`。`DataBatch` 包含不可变行、来源、请求参数、抓取时点、内容哈希、provider版本和诊断信息。provider 不得返回部分成功而隐藏失败标的；部分成功必须显式列出 `failed_items`，运行门禁据此失败。

### 3.3 字段级溯源

标准化表至少包含：

- `instrument_id`
- `effective_at`：数据对应日期或事件实际发生时间
- `available_at`：市场在何时可知该数据；财务数据使用实际披露时间
- `fetched_at`
- `source_name`
- `source_uri`
- `source_record_id`
- `payload_sha256`
- `run_id`
- 业务字段

## 4. 存储设计

```text
data/
  raw/<dataset>/<source>/<YYYY-MM-DD>/<run_id>/
  normalized/<dataset>/effective_date=YYYY-MM-DD/*.parquet
  curated/research.duckdb
  manifests/<run_id>.json
  quarantine/<run_id>/
```

- raw只追加，不原地修改；保存原始响应、请求元数据和哈希。
- normalized按数据域和有效日期分区，执行显式模式转换和去重。
- curated只从已通过门禁的normalized批次生成。
- DuckDB只保存视图、索引友好的快照和运行元数据；Parquet是主要列式文件。
- 缓存命中仍需验证哈希、来源、时效和运行配置；缓存过期或校验失败不得使用。

## 5. 有限重试和失败关闭

### 5.1 重试策略

- 每个 provider 请求最多 `3` 次总尝试，包括第一次。
- 仅对超时、连接中断、HTTP 429和5xx重试。
- 认证失败、模式变化、解析失败、4xx参数错误和数据校验失败不重试。
- 固定退避为 `1s`、`2s`；不使用无界等待。
- 每个数据集最多切换 `1` 个备用源。
- 每个数据集有 `60s` 默认墙钟截止时间，可配置但必须有上限。
- 超过上限写入 `FailureRecord` 并打开本次运行的熔断器；不得在同一运行中再次循环尝试。

### 5.2 运行状态机

```text
CREATED -> FETCHING -> NORMALIZING -> VALIDATING
VALIDATING -> READY -> COMPUTING -> EVIDENCE -> BACKTEST_LOOKUP -> REPORTING -> SUCCEEDED
FETCHING/NORMALIZING/VALIDATING -> BLOCKED_DATA
COMPUTING/EVIDENCE/BACKTEST_LOOKUP/REPORTING -> FAILED
```

只有 `READY` 可以进入 `COMPUTING`。任何必需数据集失败，整次运行停止，不调用语言模型，不输出候选评分；错误报告仍需生成。

## 6. 数据质量门禁

每个数据集在YAML中声明：必需列、主键、允许空值、最小覆盖率、最大新鲜度、数值范围、交易日历和交叉校验规则。

必需检查：

1. 模式和类型。
2. 主键唯一与重复记录内容一致。
3. 覆盖率：预期股票数、交易日数和财务期数。
4. 时效：按中国/美国交易日历和数据发布频率判断，不把节假日误判为过期。
5. OHLC关系、非负成交量、价格跳变、复权因子正值与连续性。
6. 财务会计恒等式的可用子集和披露日期合法性。
7. 股票基础信息与行情标的一致。
8. 主备源抽样差异阈值。
9. 公告/新闻内容哈希、canonical URL和转载簇。
10. 必需外部环境序列的最新可用观测。

质量结果只有 `PASS` 或 `FAIL`；生产门禁不使用可继续的 `WARN`。非必需展示字段可以记录 `INFO`，但不能参与评分。

## 7. 股票池

默认仅保留沪深A股主板代码范围，规则全部位于 `config/universe.yaml`：

- 排除创业板、科创板、北交所。
- 排除名称或状态为ST、*ST、退市整理和已退市标的。
- 最少上市 `250` 个交易日。
- 排除连续停牌达到配置天数的标的。
- 过去20日平均成交额不低于配置阈值，过去20日有效交易天数达到阈值。
- 标记停牌、涨停锁死、跌停锁死和其他无法正常成交状态。
- 股票池按每个历史日期重建，不使用当前成分回填历史。

## 8. 因子设计

### 8.1 通用规则

- 所有滚动窗口只使用 `available_at <= as_of` 的数据。
- 横截面标准化先按1%/99%分位截尾，再计算z-score和0–100百分位分数。
- 不对必需因子做均值、零值或同业中位数填充。
- 必需因子缺失时该股票从候选池排除，并保留缺失原因。
- 每个输出含 `raw_value`、`z_value`、`score`、`as_of`、`dependencies` 和 `status`。
- 权重只存在于 `config/factors.yaml`。

### 8.2 第一版公式

1. **趋势因子**
   - `0.5 * (close / MA20 - 1) + 0.5 * (MA20 / MA60 - 1)`。
   - 依赖至少60个有效交易日日线。
2. **相对强弱因子**
   - `stock_return_60d - benchmark_return_60d`。
   - 个股和基准必须使用同一截止日和复权口径。
3. **成交量和换手率因子**
   - `0.5 * z(log(volume / mean(volume.shift(1), 20))) + 0.5 * z(turnover_rate_20d_percentile)`。
   - 当日成交量不得进入过去20日均量基线。
4. **波动率和回撤因子**
   - `-0.5 * z(std(daily_return, 20) * sqrt(252)) - 0.5 * z(abs(max_drawdown_60d))`。
   - 低波动、低回撤得分更高。
5. **基本面质量因子**
   - `0.35 * z(revenue_yoy) + 0.35 * z(net_profit_yoy) + 0.30 * z(operating_cashflow / max(abs(net_profit), epsilon))`。
   - 只使用截至日已披露报告；净利润接近零导致比率失真时状态为缺失。
6. **估值历史分位因子**
   - `1 - mean(pe_ttm_percentile_3y, pb_percentile_3y)`。
   - PE为负或历史不足时不填充；可配置为只使用PB，但必须改变因子版本。
7. **行业景气因子**
   - `0.6 * industry_relative_return_60d + 0.4 * industry_fundamental_breadth`。
   - breadth为行业内营收和利润同比为正公司的占比，按实际披露日计算。
8. **公告和事件催化因子**
   - 程序按已验证事件计算：A级 `1.0`、B级 `0.8`、两个独立C级 `0.6`；D级和无法验证事件为 `0`。
   - 30日线性时间衰减，重大反方证据按配置扣分；语言模型不能直接给最终数值。

综合分数为各因子分数的YAML权重加权平均。任一配置为必需的因子缺失，股票不得进入候选池。

## 9. Evidence Gate

### 9.1 证据等级

- A：交易所公告、公司正式公告、监管披露。
- B：公司官方回应、正式会议材料、政府或权威机构文件。
- C：可靠财经媒体的独立报道。
- D：财经博主、社交媒体、论坛和未经确认传闻。

### 9.2 验证算法

1. 规范化URL、标题、实体、事件时间和正文哈希。
2. 使用内容指纹和来源关系形成转载簇；同一簇只算一个来源。
3. A或B可成为核心证据。
4. 没有A/B时必须存在两个不同所有权、不同转载簇的C级来源。
5. D只生成搜索线索。
6. 每个正方事件必须执行反方查询并保存查询结果。
7. 重大事件无法验证时状态为 `UNVERIFIED`，催化得分为0。
8. 输出等级、可信度、原始来源、发布时间、事件时间、验证状态、独立来源数和矛盾列表。

语言模型只接收已通过数据门禁的文本和元数据，用于分类、实体/事件抽取、产业链关系、矛盾归纳和自然语言解释。所有输出使用结构化模式校验；校验失败则Evidence Gate失败。

## 10. 美国市场和期货风险环境

该模块是每日运行的必需上下文，但不生成境外交易信号。

- 美国行业：官方经济/行业发布、SEC披露和可靠媒体证据，输出行业变化、可能影响的A股产业链和反方证据。
- 美国股市趋势：主要指数、波动率、利率和信用环境的程序计算标签，如趋势向上/下、波动率上升/下降、金融条件收紧/放松。
- 期货资源：CFTC COT持仓变化、公开期限结构、基差或商品指数；只输出拥挤度、风险偏好和原材料压力标签。
- 数据按各自发布日历判断最新可用值。周频COT在发布窗口内不因不是当天数据而失败。
- 任一配置为必需的外部序列缺失或过期，整次运行在推理前终止。

## 11. 自研回测

### 11.1 时间与成交模型

- 信号在交易日T收盘且数据完成发布后生成，默认在T+1开盘尝试成交。
- 同日买入的股份不得同日卖出；持仓记录保存可卖数量以实现T+1。
- 交易单位默认为100股。
- 信号计算用可配置复权口径；成交和现金账本使用原始价格与公司行动调整。
- 没有行情、停牌或锁死涨跌停时订单不成交并记录原因。
- 涨跌停规则按板块、证券状态和生效日期配置；主板普通股与ST分别处理。
- 仅触及涨跌停不等于无法成交；默认只有一字板或缺乏可成交价格时拒单。

### 11.2 成本

- 买入佣金、卖出佣金、最低佣金、卖出印花税和滑点均为YAML配置。
- 成本规则带生效日期，以支持历史税费变化。
- 滑点以成交方向调整价格，不允许产生优于原始成交价的结果。

### 11.3 防偏差

- 财务、公告和新闻使用 `available_at` 而非报告期或事件日期。
- 历史证券主表保存上市、退市、ST、板块和可交易区间。
- 任何查询必须带 `as_of`；核心查询API拒绝无截止时点调用。
- 单元测试通过向未来追加极端数据验证既有信号不变。
- 滚动回测明确训练、验证、测试窗口；第一版无学习模型，训练期用于参数选择与敏感性分析。

### 11.4 输出指标

年化收益、最大回撤、夏普比率、胜率、盈亏比、换手率、交易次数、相对基准超额收益、牛/熊/震荡阶段表现、参数敏感性、样本内/样本外结果和滚动回测结果。

实验账本保存配置哈希、代码版本、数据manifest、开始结束时间、参数、指标、交易明细、失败状态和失败原因。失败实验不可删除或被最优参数覆盖。

## 12. 每日报告

候选最多5只，排序先满足硬门禁，再按综合分数。结论只有：

- `观察`
- `等待确认`
- `进入候选池`
- `不推荐`

报告顺序：

1. 结论和运行状态。
2. 数据完整性、快照和更新时间。
3. 市场环境，包括中国市场、美国行业/股市和期货风险。
4. 行业评分与排名。
5. 候选股票和因子明细。
6. 关键证据、反方证据和验证状态。
7. 逻辑失效条件。
8. 历史同类信号表现。
9. 风险提示。

JSON是规范输出，Markdown和CSV从同一JSON模型渲染。门禁失败时JSON状态为 `BLOCKED_DATA`，候选数组为空；Markdown明确显示“不推荐”；CSV仅包含运行错误行。

## 13. CLI 和手动工作流

统一入口为 `python -m a_share_research.cli`，并提供Makefile映射：

```text
init-db                 初始化目录和DuckDB模式
update-data             增量获取并保存原始/标准化数据
validate-data           运行完整数据门禁
compute-factors         从READY快照计算因子
run-evidence-gate       验证公告和新闻事件
rank-industries         生成行业评分
select-candidates       生成最多5只候选
backtest                运行单次回测
walk-forward            运行滚动回测和敏感性分析
daily-report            串联全部阶段并输出三种格式
```

每条下游命令都验证上游manifest状态，不能通过手工跳过门禁。

## 14. 安全和配置

- 密钥只从环境变量读取；配置文件只保存环境变量名。
- 日志脱敏URL查询参数、Authorization、Cookie和token。
- HTTP请求有明确User-Agent、连接/读取超时、速率限制和有限重试。
- 原始内容视为不可信输入，HTML转文本并限制大小；语言模型提示中隔离外部文本，防止提示注入改变系统规则。
- 不实现任何券商、交易或下单接口。

## 15. 测试策略

- 单元测试：数据模型、重试、质量规则、股票池、每个因子、证据规则、撮合、成本和报告。
- 集成测试：以 `TEST_ONLY` 固定夹具执行完整流水线，不访问网络。
- 数据质量测试：缺列、重复、过期、部分标的失败、跨源冲突和复权异常必须阻断。
- 防未来函数测试：追加未来行情、未来财务或晚披露公告后，过去信号不变。
- 金样回测：手算少量交易日，覆盖T+1、停牌、涨跌停、费用、滑点和无法成交。
- 可选Qlib对照：只用于相同简化规则下比较收益曲线，不进入生产依赖。
- 生产冒烟测试：仅在显式环境变量允许时访问真实免费源；失败不影响离线测试结论，但不得声称生产数据可用。

## 16. 分阶段交付

1. 文档、项目骨架、配置和统一CLI。
2. 数据接口、Parquet/DuckDB存储、有限重试和质量门禁。
3. 股票池和八类基础因子。
4. 自研回测、实验账本、防未来函数和滚动评估。
5. Evidence Gate。
6. 美国市场、行业与期货风险上下文。
7. 行业排名、候选决策和三格式报告。
8. 手动命令稳定性验证。
9. 最后创建 `.agents/skills/a-share-daily-research/SKILL.md`。

每阶段运行相关测试、全量测试、静态检查和工作区变更审查；测试失败时该阶段保持未完成。

## 17. 已知限制

- 免费公共源可能限流、变更模式或中断；系统选择停止而不是维持报告连续性。
- 第一版的新闻覆盖取决于可访问公开来源，无法保证穷尽所有报道。
- 第一版只近似日线级成交，不模拟盘口队列和日内流动性。
- 板块和历史交易制度复杂，必须依靠带生效日期的配置和金样持续扩充。
- 完全自研回测增加正确性风险，因此金样、防未来函数、失败实验留档和可选Qlib对照是发布门禁。
- 生产密钥型数据源不在第一版默认配置内，但统一provider接口允许后续添加。

## 18. 设计决策记录

- 数据源：免费公共源优先。
- 项目：新建独立目录，不复用不符合规则的旧数据实现。
- 架构：用户选择完全自研生产运行时。
- Qlib：不进入生产；可选用于测试对照。
- 境外资产：仅为A股研究提供外部环境和风险资源。
- 审批：用户授权后续阶段自动审批，不再逐阶段询问。
- Git：本任务不创建提交。
