# A 股买点监控

这是一个本地行情监控程序，只做三件事：

- 每 60 秒拉取指定 A 股实时行情，优先走东方财富小范围股票接口，AKShare 全市场接口作为 fallback。
- 每 5 分钟在控制台打印一次所有监控股票汇总。
- 价格进入买入区间或跌破计划区间时，先经过资金风控过滤，再通过企业微信群机器人提醒人工确认。

程序不会自动下单，不会连接券商账户，也不会执行任何交易动作。

## 安装和运行本地 Web 版

建议使用 Python 3.11。

```bash
cd a_stock_watch
pip install -r requirements.txt
cp .env.example .env
python app.py
```

如果你的系统没有 `python` 命令，可以改用：

```bash
python3 app.py
```

启动后打开：

```text
http://127.0.0.1:16666
```

这个本地 Web 版会自动启动后台监控线程。页面每 5 秒刷新一次状态；后台仍然按交易时段每 60 秒拉取行情，并按 5 分钟生成一次行情快照。

顶部 `停止监控` 只会停止后台行情轮询和提醒，不会关闭网页服务；停止后可以点 `启动监控` 恢复。

旧的命令行版仍可运行：

```bash
python main.py
```

## 配置企业微信群 webhook

1. 在企业微信群里添加「群机器人」。
2. 复制机器人 webhook 地址，形如：

   ```text
   https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxx
   ```

3. 可以继续编辑 `.env`：

   ```env
   WECHAT_WEBHOOK_URL=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=你的key
   ```

本地 Web 版也支持在顶部 `配置中心` 的 `Webhook` 页签里保存企业微信群机器人地址。网页保存后会写入 `settings.json`，优先级高于 `.env`。

注意：当前版本会把 webhook 明文保存在 `settings.json`。公网部署时必须配置网页登录账号密码，并建议放在 HTTPS 反向代理后面。

## 修改买入区间

可以在顶部 `配置中心` 的 `新增/修改` 页签新增或覆盖股票，也可以直接编辑 `config.yaml`：

```yaml
- name: "沪电股份"
  code: "002463.SZ"
  buy_low: 142.0
  buy_high: 144.0
  shares: 100
```

其中：

- `buy_low` 是计划买入区间下沿。
- `buy_high` 是计划买入区间上沿。
- `shares` 用来估算提醒里的计划金额。
- `enabled: false` 可以临时关闭某只股票监控。

如果只想输入一个价格，可以在网页表单里只填 `单价`，程序会把 `buy_low` 和 `buy_high` 都设成这个价格。

`配置中心` 的 `监控股票` 页签每行有 `修改` 和 `删除`。点击 `修改` 会把该股票回填到 `新增/修改` 页签，改完点 `保存股票` 即可覆盖原配置；点击 `删除` 前会二次确认。

## 批量导入

`配置中心` 的 `批量导入` 页签默认使用 JSON，也兼容 CSV 和 YAML。

JSON 示例：

```json
{
  "watchlist": [
    {
      "name": "沪电股份",
      "code": "002463.SZ",
      "buy_low": 142,
      "buy_high": 144,
      "shares": 100,
      "type": "AI_PCB核心",
      "note": "优先买入"
    }
  ]
}
```

CSV 示例：

```csv
name,code,buy_low,buy_high,shares,type,note
沪电股份,002463.SZ,142,144,100,AI_PCB核心,优先买入
长江电力,600900,21.5,21.8,200,防守仓,防守配置
```

YAML 示例：

```yaml
watchlist:
  - name: "沪电股份"
    code: "002463.SZ"
    buy_low: 142
    buy_high: 144
    shares: 100
    type: "AI_PCB核心"
    enabled: true
```

导入时可以选择追加到当前列表，也可以勾选 `替换当前监控列表`。

## 资金风控过滤器

价格进入买入区间只代表进入观察范围，不代表可以买入。程序会继续判断，页面上会显示中文状态：

- 大盘主力资金风险：低风险 / 中风险 / 高风险 / 未知
- AI 核心票板块风险：板块低风险 / 板块中风险 / 板块高风险 / 板块未知
- 个股承接：是否继续创新低、是否低于分时均价、5 分钟跌幅是否扩大

最终提醒状态：

- 买点确认：价格进区间，市场风险未恶化，板块未高风险，个股出现承接。
- 观察提醒：价格进区间，但市场/板块/个股承接仍有分歧，只提醒观察。
- 风险拦截：价格进区间，但市场或板块高风险，禁止机械买入。
- 跌破风险：价格跌破区间下沿，提示可能是下跌中继，需要人工复盘。

如果资金数据获取失败，市场风险会显示为“未知”，程序不会发送买点确认，只会降级为观察提醒。

AI 核心观察票配置在 `config.yaml` 的 `ai_core_watch`：

```yaml
ai_core_watch:
  - "601138.SH"
  - "002463.SZ"
  - "603228.SH"
  - "002130.SZ"
  - "300394.SZ"
  - "300502.SZ"
  - "300308.SZ"
  - "300476.SZ"
  - "688256.SH"
  - "600183.SH"
```

## 沪电已买后关闭景旺电子提醒

景旺电子配置里有：

```yaml
depends_on_not_bought: "002463.SZ"
```

它表示只有 `position_state.json` 里沪电股份 `002463.SZ` 的 `bought=false` 时，景旺电子才会发送提醒。

如果沪电股份已经买入，编辑 `position_state.json`：

```json
{
  "002463.SZ": {
    "name": "沪电股份",
    "bought": true
  }
}
```

保存后重启程序，景旺电子即使进入买入区间也会跳过提醒。

本地 Web 版可以直接在 `配置中心` 的 `持仓状态` 页签里点击 `标记已买` 或 `标记未买`。

如果第一次运行时没有 `position_state.json`，程序会自动生成默认文件。

## 查看 5 分钟行情打印

程序运行后，交易时段内每 5 分钟会在控制台打印类似内容，并在网页 `时间段快照` 区域自动展示：

```text
========== A股监控行情汇总 ==========
2026-06-22 10:05:00 | 002463.SZ | 沪电股份 | 最新价 143.20 | 涨跌幅 +1.25% | 成交额 12.30亿 | 买入区间 142.00-144.00 | 状态 进入区间
```

页面和快照字段含义：

- 进入区间：价格进入买入区间，只代表进入观察范围，最终提醒由资金过滤后的状态决定。
- 等待回落：价格高于买入区间，只打印日志，不提醒。
- 跌破区间：价格低于买入区间，会发送一次风险提醒，强调跌破计划区间，不能机械买入，需要人工复盘。
- 无价格 / 无行情：本轮行情没有可用价格或没有找到该股票。
- 大盘资金风险：低风险 / 中风险 / 高风险 / 未知。
- AI 板块风险：板块低风险 / 板块中风险 / 板块高风险 / 板块未知。
- 最终提醒状态：买点确认 / 观察提醒 / 风险拦截 / 跌破风险 / 无提醒。

提醒去重规则：同一只股票同一天同一类提醒只发送一次，状态记录在 `alert_state.json`。

本地 Web 版还会把 5 分钟行情快照追加写入 `market_snapshots.jsonl`，便于后续排查。

## 交易时段

程序只在 Asia/Shanghai 时区的 A 股交易时段拉取行情：

- 09:30-11:30
- 13:00-15:00

非交易时段会等待下一轮检查。当前版本按周一到周五和上述时间段判断，不内置节假日交易日历。

## AKShare 字段变化处理

实时行情现在优先使用东方财富小范围股票列表接口，只拉取 `watchlist` 和 `ai_core_watch` 里的股票，避免每分钟拉全市场分页接口导致远端断开连接。`ak.stock_zh_a_spot_em()` 保留为 fallback。

fallback 的 AKShare 全市场接口依赖这些字段：

- `代码`
- `名称`
- `最新价`
- `涨跌幅`
- `成交额`

如果 AKShare 返回字段名发生变化，程序会报出缺失字段，并打印本次返回的实际字段名，方便你按新接口调整代码。

## 重要边界

这个程序只是提醒工具，不是交易系统。收到企业微信消息后，仍需要人工确认指数、板块、成交量、分时承接和自己的仓位，再决定是否下单。

## 公网服务器部署

公网部署时不要裸奔，至少配置网页登录账号密码。你的监控域名建议使用：

```text
gp.flylin.com.cn
```

推荐做法是用 Nginx/Caddy 做 HTTPS 反向代理，应用只监听本机。编辑 `.env`：

```env
APP_HOST=127.0.0.1
APP_PORT=16666
APP_USERNAME=你的登录用户名
APP_PASSWORD=换成强密码
WECHAT_WEBHOOK_URL=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=你的key
```

启动：

```bash
cd a_stock_watch
pip install -r requirements.txt
python app.py
```

建议用 Nginx 做 HTTPS 反向代理，把外部域名代理到：

```text
http://127.0.0.1:16666
```

Nginx 里使用：

```nginx
server_name gp.flylin.com.cn;
```

HTTPS 证书使用：

```bash
sudo certbot --nginx -d gp.flylin.com.cn
```

如果不做反向代理、要直接开放端口，则把 `.env` 改成：

```env
APP_HOST=0.0.0.0
APP_PORT=16666
APP_USERNAME=你的登录用户名
APP_PASSWORD=换成强密码
```

同时需要在云服务器安全组/防火墙里放行 `16666`。公网访问时浏览器会弹出账号密码框，这是 Basic Auth，用来保护页面、watchlist、持仓状态和 webhook 配置。不要使用 `6666` 作为浏览器访问端口，Chrome 会拦截为不安全端口。
