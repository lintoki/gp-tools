# 短线量化策略工作台

本程序是本地短线策略研究工具，只做三件事：

- 在页面上选择策略并触发当前信号输出。
- 在页面上选择策略和历史周期并触发回测。
- 输出策略质量、T+1 评估明细、净值曲线和后续操作建议。
- 保留最近几次回测结果，避免本地文件无限增长。

它不会连接券商账户，也不会自动下单。页面输出的信号只做提醒和复盘参考。

## 技术选型

底层回测框架选用 Backtrader。它是 Python 量化回测领域使用量较高的开源框架，支持策略、数据源、手续费、滑点、订单模拟和分析器。当前项目在 Backtrader 外面封装了一层策略注册中心，后续新增策略只需要增加一个策略模块，页面会自动出现。

## 启动

```bash
cd shortline_quant
python3 -m pip install -r requirements.txt
python3 app.py
```

然后打开：

```text
http://127.0.0.1:17777
```

## 数据获取与实时筛选

页面不要求输入股票代码，也不提供股票缓存开关。每次操作都由系统拉取数据：

- `触发策略信号`：请求即触发；先从实时涨幅榜做粗筛，只保留沪深主板、非 ST、非停牌、当前涨幅 3%-5% 附近的股票。不会对全市场 5000 多只股票逐个请求详情。
- 粗筛后只对少量候选拉取分时、均线等详情；近 20 日涨停次数从本地涨停缓存读取，不在尾盘逐只请求 20 日历史。
- `运行回测`：按页面选择的历史周期拉取 A 股历史行情；每个 T 日先按历史日线涨幅榜口径筛出沪深主板、非 ST、涨幅 3%-5% 的候选，再执行策略硬条件，T+1 验证结果。

数据源优先走 AKShare；如果单股历史行情被远端断开，会 fallback 到腾讯历史 K 线接口。实时信号不走逐股全市场历史扫描；回测不再固定取股票列表前 80 只，而是按每个历史交易日的涨幅榜口径生成候选池。

### 涨停缓存

收盘后可以刷新当天涨停池缓存：

```bash
curl -X POST http://127.0.0.1:17777/api/limit-up-cache/refresh \
  -H 'Content-Type: application/json' \
  -d '{"trade_date":"2026-06-23"}'
```

缓存文件位于 `data/limit_up_cache.csv`，字段包括：

```text
code,name,trade_date,close_price,limit_up_price,limit_up_reason
```

## 本地文件管理

系统不缓存股票行情。只保留最近几次回测结果：

每次回测按交易日横截面评估：T 日先用与实时策略一致的主板、非 ST、3%-5% 涨幅榜口径生成候选，再做策略过滤；T+1 查看这些候选是否冲高、是否收盘兑现收益。每次回测保存：

```text
summary.json       # 策略质量和核心指标
trades.csv         # T日信号和T+1评估明细
equity_curve.csv   # 净值曲线
```

默认只保留最近 5 次回测结果。旧结果会在每次保存后自动删除。

## 策略输出怎么触发

页面左侧选择策略后，有两个动作：

- `触发策略信号`：调用 `/api/signals`，系统输出 `strategy`、`timestamp`、`filters`、`candidates`、`rejections`、`decision`。候选股会带评分、动作、买入条件和次日卖出纪律；剔除股票会带阶段和原因。
- `运行回测`：调用 `/api/backtests`，必须选择历史开始日和结束日。系统按交易日横截面回测，T 日生成候选，T+1 评估是否正确，输出策略质量、信号正确率、按天正确率和净值曲线。

两者使用同一套策略参数。信号输出用于“现在是否进入观察”，回测用于“这个策略历史质量如何”。

回测周期由页面的 `开始日` 和 `结束日` 控制。系统只把这个周期内的 T 日信号纳入评估；为了能验证周期最后一天的 T+1 表现，后端实际拉取行情时会额外向后多取 10 个自然日的数据。
